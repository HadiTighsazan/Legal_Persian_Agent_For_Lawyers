# Task 1: Add `openai` Dependency & pgvector Migration

## Context

This is **Task 1 of Epic E-05 (Embedding & Vector Storage)** for the DocuChat project. The project is a Django + React/Vite application running in Docker. The PostgreSQL database already uses the `pgvector/pgvector:pg16` image (see `docker-compose.yml`), so the `vector` extension is available. The `DocumentChunk` model currently stores embeddings as `TextField` (temporary), and we need to migrate it to a proper `VECTOR(1536)` column using pgvector.

**Approach: B — Add `django-pgvector` dependency and use `VectorField` in the model.** This is cleaner for long-term maintainability since Django will manage the column type natively.

### 4. Create Migration: [`src/backend/documents/migrations/0004_add_pgvector_embedding.py`](src/backend/documents/migrations/)

Create a new migration file. Since we're using `django-pgvector`, we can use a combination of `migrations.RunSQL` (for the extension and index) and Django's auto-detected `AlterField` (for the column type change).

**The migration must:**
1. **Depend on** `('documents', '0003_add_processing_fields')`
2. **Operation 1:** `RunSQL` — `CREATE EXTENSION IF NOT EXISTS vector` (idempotent)
3. **Operation 2:** `AlterField` — Change `embedding` from `TextField` to `VectorField(dimensions=1536, null=True, blank=True)` — Django will auto-detect this when you run `makemigrations` after changing the model
4. **Operation 3:** `RunSQL` — `CREATE INDEX IF NOT EXISTS idx_chunks_embedding ON document_chunks USING ivfflat (embedding vector_cosine_ops)`

**Important considerations:**
- The `AlterField` with `VectorField` will generate SQL like `ALTER TABLE document_chunks ALTER COLUMN embedding TYPE vector(1536) USING embedding::vector`. If existing data has invalid text in the `embedding` column, this cast will fail. Since the field is `null=True, blank=True`, NULL values are fine. But any non-NULL text values must be valid vector representations (e.g., `[0.1, 0.2, ...]`).
- The `ivfflat` index requires the `vector` extension to be created first, so order matters.
- Run `python manage.py makemigrations` first to auto-generate the `AlterField` operation, then manually add the `RunSQL` operations around it.

**Expected auto-generated migration (from `makemigrations`):**
```python
# Generated by Django 4.2.16
from django.db import migrations, models
import pgvector.django


class Migration(migrations.Migration):

    dependencies = [
        ('documents', '0003_add_processing_fields'),
    ]

    operations = [
        migrations.AlterField(
            model_name='documentchunk',
            name='embedding',
            field=pgvector.django.VectorField(dimensions=1536, null=True, blank=True),
        ),
    ]
```

**Final migration (after adding RunSQL operations):**
```python
"""
Migration 0004: Add pgvector embedding support.

- Ensures vector extension exists
- Alters embedding column to VECTOR(1536) via VectorField
- Creates ivfflat index for cosine similarity search
"""
from django.db import migrations, models
import pgvector.django


class Migration(migrations.Migration):

    dependencies = [
        ('documents', '0003_add_processing_fields'),
    ]

    operations = [
        migrations.RunSQL(
            sql="CREATE EXTENSION IF NOT EXISTS vector",
            reverse_sql="DROP EXTENSION IF EXISTS vector",
        ),
        migrations.AlterField(
            model_name='documentchunk',
            name='embedding',
            field=pgvector.django.VectorField(dimensions=1536, null=True, blank=True),
        ),
        migrations.RunSQL(
            sql="CREATE INDEX IF NOT EXISTS idx_chunks_embedding ON document_chunks USING ivfflat (embedding vector_cosine_ops)",
            reverse_sql="DROP INDEX IF EXISTS idx_chunks_embedding",
        ),
    ]
```

## Rollback

If migration fails:
```bash
docker-compose exec backend python manage.py migrate documents 0003
```
This will run the `reverse_sql` operations (drop index, alter column back to text, drop extension) and revert the `AlterField`.

## Notes

- The Docker PostgreSQL image is `pgvector/pgvector:pg16` which has pgvector pre-installed.
- The `CREATE EXTENSION IF NOT EXISTS vector` is idempotent — safe to run multiple times.
- The `ivfflat` index uses `vector_cosine_ops` for cosine similarity search (matching OpenAI embeddings).
- If existing data has invalid embedding text, the `ALTER COLUMN ... USING embedding::vector` cast will fail. Consider clearing invalid data first or adding a data migration step.
- After adding `django-pgvector`, you must rebuild Docker images since `requirements.txt` changed: `docker-compose build backend celery_worker celery_beat`
