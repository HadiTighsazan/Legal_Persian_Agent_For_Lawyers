# Prompt for Code Mode — Fix Auth Login/Register "Internal Server Error"

## Context

After Docker infrastructure changes (multi-stage builds, image optimization), the frontend receives `{"error":"Internal server error"}` when attempting to login or register at `/auth/login/` and `/auth/register/` endpoints.

The error is a **500 response** generated by the generic `except Exception` blocks in:
- [`src/backend/users/views.py`](src/backend/users/views.py:134-140) — `register_view`
- [`src/backend/users/views.py`](src/backend/users/views.py:252-258) — `login_view`

## Investigation Results

- ✅ **`.env` file EXISTS** at project root with valid values
- ✅ `DJANGO_SECRET_KEY` is set (not empty)
- ✅ `DATABASE_URL` is set correctly
- ✅ `DJANGO_ALLOWED_HOSTS` includes `backend,nginx`
- ✅ `DJANGO_CORS_ALLOWED_ORIGINS` includes `http://localhost:5173`

## Root Cause (Confirmed)

**Migrations have NOT been applied.** The backend Dockerfile at [`docker/backend/Dockerfile`](docker/backend/Dockerfile) was rewritten for multi-stage builds and does NOT include a `migrate` step. The container starts Gunicorn directly without ensuring database tables exist.

When `register_view` or `login_view` tries to access the `users` table (which doesn't exist), Django raises a `ProgrammingError`, which is caught by the generic `except Exception` block, returning `{"error":"Internal server error"}`.

## Task Instructions

### Step 1: Create Entrypoint Script

Create `docker/backend/entrypoint.sh`:

```bash
#!/bin/sh
set -e

echo "Running database migrations..."
python manage.py migrate --noinput

echo "Collecting static files..."
python manage.py collectstatic --noinput

echo "Starting Gunicorn..."
exec gunicorn config.wsgi:application --bind 0.0.0.0:8000 --workers 3
```

### Step 2: Update Backend Dockerfile

Modify [`docker/backend/Dockerfile`](docker/backend/Dockerfile):
- After `COPY src/backend /app/`, add:
  ```dockerfile
  COPY docker/backend/entrypoint.sh /app/entrypoint.sh
  RUN chmod +x /app/entrypoint.sh
  ```
- Change `CMD` to `ENTRYPOINT`:
  ```dockerfile
  ENTRYPOINT ["/app/entrypoint.sh"]
  ```

### Step 3: Rebuild and Test

```bash

docker-compose down

# Rebuild backend image
docker-compose build backend

```
